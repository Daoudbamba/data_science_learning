# Devenir Data Scientist en 12 mois

## Objectif
Suivre une feuille de route claire pour acquérir les compétences clés d’un Data Scientist en 12 mois.

## Plan mensuel

### Mois 1 — Python & environnement
- Installer un environnement de travail (VS Code, Git, Jupyter).
- Maîtriser les bases Python : types, boucles, fonctions, modules.

### Mois 2 — Statistiques fondamentales
- Statistiques descriptives, probabilités, distributions.
- Tests d’hypothèses et intervalles de confiance.

### Mois 3 — SQL & manipulation de données
- Requêtes SQL (SELECT, JOIN, GROUP BY, window functions).
- Nettoyage de données et jointures avec pandas.

### Mois 4 — Data visualization
- Visualisations avec Matplotlib / Seaborn.
- Storytelling et choix de graphiques adaptés.

### Mois 5 — Machine Learning supervisé
- Régression linéaire/logistique, k-NN, arbres.
- Métriques de classification et de régression.

### Mois 6 — Machine Learning avancé
- Random Forest, Gradient Boosting, XGBoost.
- Validation croisée, tuning d’hyperparamètres.

### Mois 7 — Feature engineering
- Encodage, normalisation, gestion des valeurs manquantes.
- Pipelines reproductibles avec scikit-learn.

### Mois 8 — Projets de bout en bout
- Réaliser 1 à 2 projets complets (de la donnée brute au modèle).
- Documenter les hypothèses, résultats et limites.

### Mois 9 — Deep Learning (initiation)
- Réseaux de neurones, TensorFlow ou PyTorch.
- Cas d’usage simples : image, texte, séries temporelles.

### Mois 10 — Déploiement & MLOps
- API de modèle (FastAPI/Flask), Docker.
- Suivi de modèles, versioning, monitoring.

### Mois 11 — Spécialisation
- Choisir un domaine : NLP, Computer Vision, Time Series, Reco.
- Approfondir avec un projet orienté métier.

### Mois 12 — Portfolio & préparation emploi
- Consolider un portfolio GitHub (3-5 projets solides).
- Préparer CV, études de cas, entretiens techniques.

## Livrables attendus après 12 mois
- Solides bases en Python, SQL, stats et machine learning.
- Portfolio démontrant des projets concrets.
- Capacité à passer de la donnée brute à une solution déployable.
